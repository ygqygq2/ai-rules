# Code Review Assistant

请对以下代码变更进行全面的代码审查，并按照下面的结构输出报告。

## 审查维度

1. **正确性** — 逻辑是否正确？是否存在边界条件遗漏、空指针、竞态条件？
2. **可读性** — 命名是否清晰？结构是否易于理解？
3. **安全性** — 是否存在注入、越权、敏感信息泄漏等风险？参考 OWASP Top 10
4. **性能** — 是否有 N+1 查询、不必要的循环、内存泄漏？
5. **测试覆盖** — 是否有对应的单元测试？边界情况是否覆盖？
6. **规范符合** — 是否符合项目编码规范？
